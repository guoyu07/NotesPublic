用MSCS以及CA的SurviveIT实现过SQL的双机热备
(作者  _rambo)
Microsoft? SQL Server? 2000 故障转移群集允许系统自动故障服务器中的 SQL Server 实例处理转换到工作服务器，从而提高了服务器的可用性。例如，SQL Server 实例可以快速地将数据库服务还原到 Web 站点或企业网络，即使运行该实例的服务器发生故障。SQL Server 2000 基于 Windows NT? 4.0 和 Windows? 2000 中的 Microsoft 群集服务 (MSCS) 的故障转移群集功能实现故障转移群集。

SQL Server 2000 使用的 MSCS 故障转移群集类型由多台服务器计算机组成（在 Windows NT 4.0 上为两个，在 Windows 2000 Datacenter Server 上最多可以为四个），它们共享一套公用群集资源，如磁盘驱动器。群集中的每台服务器都称为一个节点。每台服务器（或节点）与网络连接，并且各节点间可以相互通讯。每一节点都运行同一版本的 MSCS。

在故障转移群集中，共享资源被收集到群集组内。例如，如果故障转移群集有四个共享磁盘驱动器，其中的两个驱动器可以收集到一个群集组内，而另两个驱动器可以收集到另一个群集组内。每一群集组都由故障转移群集中的某个节点拥有，不过这种所有权可以在节点间转让。

应用程序可以安装在故障转移群集中的节点上。这些应用程序通常是服务器应用程序或用户通过网络连接访问的分布式 COM 对象。应用程序的可执行文件和其它资源通常存储在节点所拥有的一个或多个群集组内。每一节点都可以在其上安装多个应用程序。

故障转移群集节点定期相互发送称为心跳信号消息的网络消息。如果 MSCS 软件检测到群集中某个节点丢失了心跳信号，则将该服务器视为故障服务器。MSCS 然后自动将该节点的群集组和应用程序资源转移给网络中的其它节点。当任何特定的节点发生故障时，群集管理员指定将群集组转移到的替换节点。于是，其它节点继续处理对故障服务器转移来的应用程序的用户网络请求。

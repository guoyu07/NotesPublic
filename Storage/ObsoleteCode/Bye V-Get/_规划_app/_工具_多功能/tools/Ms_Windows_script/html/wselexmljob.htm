Windows 脚本宿主  

<?job?>元素

指定用于错误处理的属性。

<?job error="flag" debug="flag" ?>

参数

error
布尔值。所有属性的默认值都为 false。如果设为 true，就允许 Windows 脚本 (.wsf) 文件中存在语法错误信息或运行时错误。
Debug
布尔值。所有属性的默认值都为 False。如果设为 true，就会启用调试。如果未启用调试，就不能为 Windows 脚本文件启动脚本调试程序。

说明

虽然大多数 Windows 脚本文件在生成过程中通常都是默默地运行，但如果能在开发 Windows 脚本 (.wsf) 文件的过程中及时了解其中的错误会是非常有用的。

示例

下面的示例将两个作业合并为一个 .wsf 文件，该文件使用两种不同的脚本语言：

<package>
<job id="DoneInVBS">
<?job debug="true"?>
<script language="VBScript">
WScript.Echo "This is VBScript"
</script>
</job>

<job id="DoneInJS">
<?job debug="true"?>
<script language="JScript">
WScript.Echo("This is JScript"); 
</script>
</job>
</package>

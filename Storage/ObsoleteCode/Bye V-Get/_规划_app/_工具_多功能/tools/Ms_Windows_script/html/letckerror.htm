Windows 脚本部件  

检查脚本部件文件中的错误

由于 Windows® 脚本部件是作为 COM 部件来使用的，它们通常是默默地运行。不过，在开发脚本部件文件时，您可能发现查找文件中的错误是很有用的。可以指定三种类型的错误检查：

  • 检查 XML 的正确性。详细信息请参阅脚本部件文件和 XML 一致性。
  • 允许通知语法和运行错误。缺省情况下，如果在脚本部件文件中发生错误，将显示一个一般的错误信息。通过指定错误通知，就可以让脚本部件语法分析器来显示该错误的详细信息。
  • 允许调试。缺省情况下不能对脚本部件使用脚本调试器。如果要启用调试，您可以通过设置断点、或使用 Stop (Microsoft® Visual Basic® Scripting Edition (VBScript)) 或 debugger (JScript) 语句来启动脚本调试器以响应错误。

设置错误选项

将错误选项指定为 XML <?component?> 处理指令的属性。

要指定错误检查

  • 在脚本部件文件的开头包括 <?component?> 处理指令（但应位于可选的 <?XML ?> 声明之后），在该指令内加入一条或多条下述属性：
    • error   将其设置为真，将显示脚本部件中的语法或运行错误的详细错误信息。
    • debug   将其设置为真将启用调试功能。如果没有启用该调试功能，则任何方式下都不能对脚本部件使用脚本调试器。

例如，下述代码启用了所有三种错误报告选项：

<?component error="true" debug="true"?>
提示   在将脚本部件部署到工作环境之前请将错误报告关闭。
